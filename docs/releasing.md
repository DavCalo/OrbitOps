# Release process

OrbitOps uses Semantic Versioning for published releases.

## Pre-release checklist

1. Ensure `main` is clean and CI is green.
2. Update the Python version in `ground_station/orbitops/__init__.py`.
3. Update the CMake project version in `onboard/CMakeLists.txt`.
4. Move relevant entries from `Unreleased` into a dated section in `CHANGELOG.md`.
5. Review protocol, deterministic link semantics, and link-event schema compatibility.
6. Run:

   ```bash
   make clean
   make bootstrap
   make verify
   ```

7. Confirm both CLIs report the same version:

   ```bash
   orbitops --version
   ./build/orbitops_sim --version
   ```

8. Run the supported public workflow:

   ```bash
   make link-demo
   ```

9. Confirm the documented supported Python versions and operating systems match CI.
10. Review known limitations and avoid flight-readiness, safety, security, or CCSDS claims that have not been independently demonstrated.

## Compatibility review

A release affecting the binary protocol, deterministic impairment decisions, or JSONL event schema requires an explicit compatibility decision.

Review:

- protocol version and golden packet vector;
- SplitMix64 and fixed draw schedule;
- golden impairment decisions;
- scheduling and duplicate ordering;
- link-event schema version;
- complete and partial log validation behavior;
- CLI option defaults and exit behavior.

## Tag and publish

Set the release version once and create an annotated or signed tag according to the maintainer's local signing setup:

```bash
VERSION=0.2.0
git tag -a "v${VERSION}" -m "OrbitOps v${VERSION}"
git push origin "v${VERSION}"
```

Create a GitHub Release from the tag and include:

- changelog summary;
- supported platforms and Python versions;
- deterministic-link and event-schema highlights;
- known security and deployment limitations;
- source archives generated by GitHub;
- checksums for any manually attached binaries.

Do not label a release as flight-ready, safety-certified, cryptographically secure, or CCSDS-compliant unless those claims are independently demonstrated.
