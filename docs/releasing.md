# Release process

OrbitOps uses Semantic Versioning for published releases.

## Pre-release checklist

1. Ensure `main` is clean and CI is green.
2. Update the Python version in `ground_station/orbitops/__init__.py`.
3. Update the CMake project version in `onboard/CMakeLists.txt`.
4. Move relevant entries from `Unreleased` into a dated section in `CHANGELOG.md`.
5. Run:

   ```bash
   make clean
   make bootstrap
   make verify
   ```

6. Confirm both CLIs report the same version.
7. Review protocol and compatibility notes.

## Tag and publish

```bash
git tag -s v0.1.0 -m "OrbitOps v0.1.0"
git push origin v0.1.0
```

Create a GitHub Release from the signed tag and include:

- changelog summary;
- supported platforms;
- known limitations;
- source archives generated by GitHub;
- checksums for any manually attached binaries.

Do not label a release as flight-ready, safety-certified, or CCSDS-compliant unless those claims are independently demonstrated.
